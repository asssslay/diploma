# Моніторинг інформаційної системи

Для оперативної діагностики та адміністрування інформаційної системи застосовано інструмент **Zabbix 7.4**. Сервер моніторингу було розгорнуто локально у **Docker Desktop**, а знімання метрик виконувалось із **задеплоєного вебзастосунку**, розміщеного на платформі **Render**. Це дало змогу оцінювати стан саме робочої інсталяції системи, а не локального середовища розробки.

Моніторинг виконувався з інтервалом **30 с** у режимі роботи **під навантаженням**. Для контролю доступності застосунку використовувався health-check URL `/healthz`, а для знімання прикладної телеметрії — серверний телеметричний endpoint задеплоєного застосунку. Отримані дані використовувалися для своєчасної діагностики відмов, деградації швидкодії та зростання споживання ресурсів.

## Контрольовані телеметричні параметри

- доступність застосунку за кодом відповіді **HTTP 200**;
- середній час обробки запиту сервером;
- обсяг оперативної пам’яті, що споживається застосунком;
- утилізація процесорного часу застосунком.

## Отримані результати

Під час моніторингу підтверджено стабільну доступність задеплоєного застосунку: health-check повертав код **HTTP 200**. У період навантаження було зафіксовано зміну всіх контрольованих метрик. Середній час відповіді зростав до **1657,29 мс**, споживання оперативної пам’яті збільшувалося приблизно від **102,09 МБ** до **117,11 МБ**, а утилізація CPU змінювалась у межах близько **0,03–0,26 %**. Це підтверджує працездатність налаштованого моніторингу та можливість використовувати його для своєчасної діагностики несправностей і контролю стану ІС.

Нижче наведено ілюстрації, отримані в інтерфейсі Zabbix для задеплоєного проєкту.
